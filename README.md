# ai201-project4-provenance-guard

Provenance Guard is a Flask backend that classifies submitted text as likely AI-generated, likely human-written, or uncertain. It combines a Groq LLM signal with a stylometric signal, returns a transparency label, supports creator appeals, rate-limits submissions, and writes a structured JSON audit log.

## Run locally

```powershell
Set-Location C:\CodePath\ai201-project4-provenance-guard
.\.venv\Scripts\python.exe app.py
```

Set `GROQ_API_KEY` before running if you want the live LLM signal. If the LLM signal is unavailable, the app degrades gracefully, records `"degraded": true`, and keeps the verdict in the uncertain band.

## Submit text

```powershell
$body = @{
  text = "Artificial intelligence represents a transformative paradigm shift in modern society. It is important to note that stakeholders must collaborate to ensure responsible deployment."
  creator_id = "test-user-1"
} | ConvertTo-Json

Invoke-RestMethod `
  -Uri http://localhost:5000/submit `
  -Method Post `
  -ContentType "application/json" `
  -Body $body | ConvertTo-Json -Depth 10
```

The response includes `content_id`, `attribution`, `confidence`, `label`, `reason`, and `timestamp`.

## Confidence thresholds and labels

`p_ai` is the internal probability that the text is AI-generated. `confidence` is the certainty of the displayed label: `max(p_ai, 1 - p_ai)`.

| `p_ai` range | Attribution | Transparency label |
|---|---|---|

## Appeals

Appeals do not re-run detection. They update the existing decision status to `under_review`, preserve the original scores, append an appeal audit entry, and return confirmation.

```powershell
$appeal = @{
  content_id = "PASTE-CONTENT-ID-HERE"
  creator_reasoning = "I wrote this myself from personal experience. I am a non-native English speaker and my writing style may appear more formal than typical."
} | ConvertTo-Json

Invoke-RestMethod `
  -Uri http://localhost:5000/appeal `
  -Method Post `
  -ContentType "application/json" `
  -Body $appeal | ConvertTo-Json -Depth 10
```

Successful response:

```json
{
  "content_id": "c0d2605c-df59-40f7-a352-7b83314d6eb7",
  "message": "Appeal received. Status updated to under review.",
  "status": "under_review"
}
```

Unknown IDs return:

```json
{ "error": "content_id not found" }
```

## Rate limiting

`POST /submit` uses Flask-Limiter with `memory://` storage and client-IP keys:

- `10 per minute`
- `100 per day`

These limits are intended to allow normal creator usage while blocking a script that floods the classifier. A real writer is unlikely to submit more than a handful of items per minute, while automated abuse will hit `10/minute` quickly; `100/day` caps sustained abuse without blocking a prolific human creator.

Rate-limit exceeded response:

```json
{ "error": "Rate limit exceeded. Try again later." }
```
